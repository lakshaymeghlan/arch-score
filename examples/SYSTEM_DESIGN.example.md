## Recommended folder structure

Current structure reads as **hexagonal**; the recommended target for a cli project is **layered**.

> Keep IO at the edges: a thin bin/ entry parses args and delegates to commands/, which call pure core/ logic. This keeps the business logic testable without spawning the CLI.

```
bin/                 # executable entry, arg parsing only
src/
  commands/          # one module per command, thin
  core/              # pure business logic, no process/IO
  adapters/          # fs, network, process boundaries
  config/
tests/
```

### Gaps vs. recommended layout

- No `commands/` directory — recommended for cli projects.
- Current layout reads as "hexagonal"; consider migrating toward "layered" for this project type.
